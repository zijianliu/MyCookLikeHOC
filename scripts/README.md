# 菜谱数据提取脚本

这个脚本用于将 `cook-book` 目录下的所有菜谱 Markdown 文件提取并上传到 Supabase 数据库。

## 功能特性

- 🔍 **自动遍历**: 递归遍历 `cook-book` 目录下的所有 `.md` 文件
- 📝 **智能解析**: 自动解析 Markdown 文件的标题、配料、步骤等信息
- 🖼️ **图片处理**: 将图片文件转换为 base64 格式存储
- 🏷️ **分类识别**: 根据文件路径自动识别菜谱分类
- 🔄 **重复检测**: 避免重复上传相同标题的菜谱
- 📊 **进度显示**: 实时显示处理进度和结果统计

## 文件说明

### extract-recipes.js
主要的数据提取和上传脚本，包含以下功能：
- 解析 Markdown 文件内容
- 图片转 base64 编码
- 上传数据到 Supabase
- 错误处理和重复检测

### verify-data.js
数据验证脚本，用于检查上传的数据：
- 统计总菜谱数量
- 按分类统计菜谱数量
- 显示样本数据
- 检查图片上传情况

## 数据库结构

脚本会在 Supabase 中创建 `recipes` 表，包含以下字段：

| 字段名 | 类型 | 说明 |
|--------|------|------|
| id | UUID | 主键，自动生成 |
| title | VARCHAR(255) | 菜谱标题 |
| category | VARCHAR(100) | 菜谱分类 |
| ingredients | TEXT | 配料信息 |
| steps | TEXT | 制作步骤 |
| cover_image | TEXT | 封面图片（base64编码） |
| image_path | VARCHAR(500) | 原始图片路径 |
| created_at | TIMESTAMPTZ | 创建时间 |
| updated_at | TIMESTAMPTZ | 更新时间 |

## 使用方法

### 1. 运行数据提取脚本

```bash
cd scripts
node extract-recipes.js
```

### 2. 验证上传的数据

```bash
node verify-data.js
```

## 注意事项

1. **图片路径**: 脚本会自动处理 Markdown 中的相对路径图片引用
2. **重复处理**: 如果菜谱标题已存在，会跳过上传避免重复
3. **错误处理**: 单个文件处理失败不会影响其他文件的处理
4. **权限设置**: 数据库表已配置适当的 RLS 策略

## 支持的 Markdown 格式

脚本支持以下 Markdown 格式：

```markdown
# 菜谱标题

![图片描述](../images/图片名.png)

## 配料
- 配料1
- 配料2

## 步骤
- 1. 步骤1
- 2. 步骤2
```

或者使用 `## 品类` 代替 `## 配料`。

## 故障排除

### 常见问题

1. **图片文件不存在**: 脚本会输出警告但继续处理
2. **Supabase 连接失败**: 检查网络连接和 API 密钥
3. **权限错误**: 确保 Supabase 项目配置正确

### 日志信息

脚本会输出详细的处理日志，包括：
- 文件处理进度
- 数据解析结果
- 上传状态
- 错误信息

## 技术栈

- **Node.js**: 运行环境
- **@supabase/supabase-js**: Supabase 客户端
- **fs/path**: 文件系统操作
- **原生 JavaScript**: Markdown 解析和数据处理
